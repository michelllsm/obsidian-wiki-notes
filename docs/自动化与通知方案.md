# 自动化与通知方案

> 目标：让用户先快速上手SKILL，再在合适时机渐进开启自动化（避免首次配置过长）。

---

## 1) 推荐原则（先手动，后自动）

| # | 原则 |
|---|------|
| 1 | **首次开箱不要求配置自动化**，只确认最小必要项（路径、命名、输出偏好）。 |
| 2 | **写完第一篇每日笔记后**，再询问是否开启"每日笔记自动化"。 |
| 3 | **累计完成 3 篇笔记后**，再询问是否开启"周复盘自动化"。 |
| 4 | 保持"双模式并存"：**手动可随时使用，自动仅作增效**。 |

---

## 2) 内置自动化（优先）

若当前平台有"内置自动化/定时任务"能力，优先使用。

### 2.1 任务A：每日笔记自动化（可选）
- 名称：`每日笔记提醒/生成`
- 频率：每日一次（建议晚间）
- 触发动作：`-day`
- 输出：`📅每日笔记/`

### 2.2 任务B：周复盘自动化（可选）
- 名称：`每周复盘`
- 频率：每周五 22:30（可改）
- 触发动作：`-week`
- 输出：`📆每周笔记/`

### 2.3 通知建议
- 手动模式：不强制通知
- 自动模式：生成成功后发送提醒

## 3) 跨平台备选方案

### 3.1 方案A：cron + 本地脚本（macOS/Linux）
```cron
# 每天 21:30 执行每日笔记任务
30 21 * * * /usr/bin/python3 /path/to/scripts/run_daily_note.py

# 每周五 22:30 执行周复盘任务
30 22 * * 5 /usr/bin/python3 /path/to/scripts/run_weekly_review.py
```

### 3.2 方案B：GitHub Actions（仓库托管场景）
```yaml
name: notes-automation
on:
  schedule:
    - cron: "30 13 * * *"   # 北京时间 21:30，每日
    - cron: "30 14 * * 5"   # 北京时间 22:30，每周五
jobs:
  run:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run notes workflow
        run: python scripts/run_notes_workflow.py
```

## 4) 常见问题

**Q：自动化会不会影响手动记录？**  
A：不会。手动与自动可并行。

**Q：一开始要不要就全配自动化？**  
A：不建议。先跑通手动，触发条件满足后再开启。

**Q：内容太长怎么办？**  
A：确认模板只启用【必填】模块。
